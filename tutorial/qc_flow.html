📝 SHOT: sq01 c05 Compositing (シーケンス 1・カット 5・合成 task) / asset: v04 (4 回目提出)

1
🖥️

佐藤 User が 部内 Review を提出

sq01 c05 v04 を完成 → shot_detail で「📤 既存 asset で QC 依頼」 → modal で 📌 Review を選択 → mention で 加藤 Lead を指定 → 送信。

🔧 システム動作

  • SHOT 関係者 thread 自動作成 (PM 田中 + 加藤 Lead + 佐藤 User)
  • task.status → qc 自動 patch
  • QC ビューア緑 button 付き本文投稿
  • Push + SSE で加藤 Lead へ配信
→ Review 依頼 機能詳細 → User 動線
2
💡

加藤 Lead が 部内 Review 判定

加藤 Lead が Push 通知で気付く → SHOT thread から ▶ QC ビューア緑 button click → asset 視聴 → Lighting 観点で問題なし → thread に「部内 OK」 と返信。

🔧 システム動作

  • qc_viewer で video 視聴 (同期プレイヤー 実時間連動)
  • thread に返信 → 佐藤 User に Push + SSE 配信
  • Lead はこの時点で「上申 (= Director QC 提出)」 も可能 (mention 必須 enforce)
→ Lead 動線 → QC ビューア機能
3
🖥️

佐藤 User が Director QC を提出

部内 OK を受けて、 同じ asset (v04) を 🔍 QC として提出 → 「桜咲く街」 project の Director (山田) は自動補完 → 送信。

🔧 システム動作

  • project.director_id 設定済 → 山田 Director を自動 participants に追加
  • Director 不在 project だった場合は mention 必須 enforce + HTTP 400 防壁
  • task.status は既に qc (Step 1 で patch 済)
  • SHOT thread に「🔍 QC 依頼」 投稿 → 山田 へ Push + SSE 配信
→ QC 依頼 機能詳細 → User 動線
4
🎬

山田 Director が QC viewer で判定

山田 Director Dashboard の 📥 受信 QC 依頼 紫 card click → qc_viewer 遷移 → 桜咲く街 sq01 c05 v04 視聴 → version 履歴で v03 と比較 → ここで 2 つの分岐 が発生。

🔧 システム動作

  • 同期プレイヤーで video 実時間連動 (旧 placeholder 廃止)
  • パンくず 4 階層: プロジェクト › 桜咲く街 › sq01 › c05 › Compositing
  • version 履歴サイドバーで過去 version 比較
  • 判定 block 活性化 (task.status=qc が条件)
→ QC ビューア 機能詳細
↙️ 分岐 A: 承認
↘️ 分岐 B: 差戻し
5A

✅ Approve (承認)

山田 Director が問題なし判定 → コメント入力 → ✅ Approve button click。

🔧 システム動作:

  • task.status → ap patch (承認済・納品 deliver はこの時点では別扱い)
  • SHOT thread に「✅ Approved」 通知
  • Push + SSE で全 participants 配信
  • 佐藤 User Dashboard でも承認済表示

🎉 sq01 c05 Compositing 承認 → 納品(Deliver)は別トリガーで実施

5B

🔁 Retake 発令

山田 Director「桜花ピンク 10% 強化希望」 → 🔁 Retake link click → director_retake_input page へ。

🔧 director_retake_input で:

  • 全体方針 textarea 入力
  • video player でマーカー保存 (0:12 桜並木)
  • 参考画像 2 枚添付 (drag&drop)
  • 期日 default = 今日 (調整可)
  • 🚀 リテイクを発行 → POST /api/bff/retakes

🔧 サーバ側:

  • task.status → qc_fb (社内 Retake feedback 専用ステータス)
  • SHOT thread に「🔁 Retake 発令」 投稿
  • 佐藤 User へ Push + SSE 配信
  • /tmp/score_retake_refs/{retake_id}/meta.json + 添付保存
→ Retake 発令 機能詳細
6
🖥️

佐藤 User が /retake_view で確認 → 修正着手

Retake 通知 Push toast → SHOT thread →「▶ Retake 確認」 button → /retake_view/{shot_id}/{task_id} 遷移 → 対象ビューア + マーカー TC click でジャンプ + コメント card + 参考素材 inline → 修正方針把握 → 修正作業 → v05 で Step 3 へ戻る。

🔧 /retake_view 表示内容

  • 🎯 対象 asset video + TC overlay + マーカー click で video シーク
  • 📝 全体方針 (Director 入力 direction 全文)
  • 📍 メタ (優先度 / 期日 / 発令者名 / 発令日時)
  • 💬 コメント card click → video seek + play
  • 📎 参考素材 inline (画像拡大 / 動画 player / URL link)
  • 💬 task thread を見る link (messages 連動)

⤴️ 修正完了したら Step 3 (Director QC 提出) に戻る → Approve まで loop

→ Retake 確認 page 詳細

📊 flow 全体まとめ

「桜咲く街」 sq01 c05 Compositing の QC/Review 1 サイクルは、 4 role × 6 step で構成され、 通常 1〜3 日 で完了します (Retake が複数回発生する場合は延長)。

Score は各 step で SHOT thread・Push・SSE・画面 badge の三重通知で 連絡漏れを防ぎ、 task.status (mk → wip → qc → ap → deliver) で 公式 ステータスを track します (Retake 発生時は qc_fb を経由。deliver = 納品は Approve とは別トリガー)。 PM 田中は全 SHOT thread に自動参加し、 進捗を ダッシュボードで横断把握します。
