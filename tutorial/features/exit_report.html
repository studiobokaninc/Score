🚪 退勤報告

📘 この機能の目的

本日の作業内容・明日の予定・引き継ぎ事項を提出。 翌朝の routine で前日提出済 check に使用。

📸 画面

📍 画面例: 佐藤 User が「桜咲く街 sq01 c05 v04 提出済・blocker なし・明日は sq02 c01 着手」 と記入し退勤。 山田 Director が翌朝 routine で内容確認可能。

✨ 主な機能

  • 当日提出 / 前日分提出 — mode=previous で前日業務日分も提出可
  • blocker (止まった要因) 記入 — 翌日対応の参考
  • handover (引き継ぎ) — チームメンバーに自動共有
  • next_priority (明日の優先) — routine で表示
  • 未提出時ログアウト確認 dialog — 提出忘れ防止
  • JST 業務日 boundary (5am 区切り) — 殿御命 2026-06-08 厳格化

🌸 使用例 — 架空 project「桜咲く街」

佐藤 User が「桜咲く街 sq01 c05 v04 提出済・blocker なし・明日は sq02 c01 着手」 と記入し退勤。 山田 Director が翌朝 routine で内容確認可能。

🛠 操作方法

  1. サイドメニュー🚪 → 退勤報告
  2. 本日作業 + 明日予定 + 引き継ぎ記入
  3. 提出 → 通知が thread 自動投稿
  4. 💡 前日未提出時はログイン時に強制表示
