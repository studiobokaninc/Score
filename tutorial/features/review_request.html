📌 Review 依頼

📘 この機能の目的

User が部内 (Lead) に Review 依頼を提出。Director 判定前段階の意見集約。 依頼 button 1 click で SHOT 関係者全員に thread + Push 自動配信される。

📸 画面 (2 枚)

📤 [依頼する側] User → Review 提出 modal

User (佐藤) が shot_detail で「📤 既存 asset で QC 依頼」 → modal で「📌 Review」 選択 + 加藤 Lead を mention + コメント記入 → 送信

📥 [される側] Lead が SHOT thread で受信

加藤 Lead が SHOT 関係者 thread で「📌 Review 依頼」 + 階層 title + asset サムネ + QC ビューア緑 button を受信 → ▶ click で qc_viewer 遷移

📍 シナリオ: 佐藤 User が桜咲く街 sq01 c05 v03 完成 → 「📌 Review」 選択 + 加藤 Lead mention → 部内意見集約 → 加藤 OK後 山田 Director に QC 提出。

✨ 主な機能

  • 提出種別 = Review — 部内レビュー目的・Director 判定前の意見集約
  • SHOT 関係者 thread 自動作成 — PM + Director + Lead + assignees + mention で構成
  • mention 主体 — Director 自動補完なし。mention で送信先を明示指定
  • 階層 title — proj/SEQ/SHOT/task の 4 階層を本文先頭に明記
  • QC ビューア link 自動付与 — 本文末尾に /qc/{shot_id}?task_id&asset_id 緑ボタン化
  • Push + SSE 配信 — 全 participants に通知 (sender 除外・mention 含 sender は配信)
  • task status → review patch — 殿御命 2026-06-08 自動 patch 追加

🌸 使用例 — 架空 project「桜咲く街」

佐藤 User が桜咲く街 sq01 c05 v03 完成 → 「📌 Review」 選択 + 加藤 Lead mention → 部内意見集約 → 加藤 OK後 山田 Director に QC 提出。

🛠 操作方法

  1. shot_detail で対象 asset card → 📤 既存 asset で QC 依頼 → modal で「📌 Review」 選択
  2. mention 候補から加藤 Lead を選択 (Director 不在時必須)
  3. 補足コメント (任意) → 📤 送信
  4. 受信側: msg → SHOT thread → ▶ QC ビューア確認 → 部内コメント返信
