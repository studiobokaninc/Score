🎯 QC ビューア

📘 この機能の目的

提出 asset を Before/After 並列比較・version 履歴サムネ・Director 判定 (Approve/Retake)・Lead 上申 を統合した QC 専用画面。

📸 画面 (5 枚)

① メイン view (単独 player + 判定 block)

Director がメイン画面で asset (sq01 c05 v04) を視聴している状態 — パンくず + メイン video player + 同期 timeline + 右 sidebar に version 履歴 + 判定 block (Approve/Retake)

② Before/After 比較 view

version 履歴から v03 (左 Before) と v04 (右 After) を並列再生して比較

③ version 履歴サイドバー

過去 version サムネが時系列で並び、click で切替・比較対象選択

④ Approve button + コメント入力

Director が ✅ Approve + コメント入力 直前の状態 (task=completed patch 直前)

⑤ 🔁 Retake link

判定 block 内「🔁 Retake」 link 部分 (click で /director_retake_input 遷移)

📍 シナリオ: 山田 Director が桜咲く街 sq01 c05 v04 を qc_viewer で視聴 → 同期プレイヤーで実時間再生 → version 履歴で v03 と比較 → Lighting 微調整必要 → 🔁 Retake click で director_retake_input へ。

✨ 主な機能

  • asset 自動再生 — URL ?asset_id=N で指定 asset を mp4/mov player で表示
  • パンくず — プロジェクト › 桜咲く街 › sq01 › c05 › Compositing › QC確認 (4 階層 full name)
  • 同期プレイヤー 実時間連動 — video.timeupdate で sync-timeline 同期 (殿御命 2026-06-08)
  • Director / PM 判定 block — ✅ Approve + コメント / 🔁 Retake link
  • Lead / Lighting Lead 上申 block — 📤 Director QC へ上申 (mention 必須対応)
  • SHOT/task thread link — ヘッダー右上「💬 c05 Compositing task thread」 で messages 連動
  • version 履歴サイドバー — 過去 version サムネ click で切替・比較モード対応

🌸 使用例 — 架空 project「桜咲く街」

山田 Director が桜咲く街 sq01 c05 v04 を qc_viewer で視聴 → 同期プレイヤーで実時間再生 → version 履歴で v03 と比較 → Lighting 微調整必要 → 🔁 Retake click で director_retake_input へ。

🛠 操作方法

  1. msg の ▶ QC ビューア緑 button click → /qc/{shot_id}?task_id&asset_id
  2. メイン player で asset 確認 + version 履歴 click 切替
  3. Director: コメント入力 → ✅ Approve → SHOT thread に「✅ Approved」 通知 + task=completed
  4. Director: 🔁 Retake → /director_retake_input?shot_id&task_id 遷移 → リファレンス添付可
  5. Lead: 📤 Director QC へ上申 → mention 必須 → SHOT thread 通知
