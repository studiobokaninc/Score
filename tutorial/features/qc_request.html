🔍 QC 依頼

📘 この機能の目的

User が Director 直接 Approve/Retake 判定を仰ぐ QC 依頼を提出。部内 OK 前提で Director に上申。

📸 画面 (2 枚)

📤 [依頼する側] User → QC 提出 modal

User (佐藤) が「🔍 QC」 選択時 — project.director 設定済なら 山田 Director が自動 mention 補完 (Director 不在時のみ mention 必須 enforce)

📥 [される側] Director の Dashboard 紫 card

山田 Director Dashboard の📥 受信 QC/Review 依頼 section に紫 card 表示 → 「🎯 QC ビューアで確認」 button で qc_viewer 即遷移

📍 シナリオ: 佐藤 User が桜咲く街 sq01 c05 v04 部内 Review OK 後、「🔍 QC」 選択 → 山田 Director 自動 mention → Director Dashboard 紫 card で気付き → qc_viewer で判定。

✨ 主な機能

  • 提出種別 = QC — 部内 OK 済前提・Director 判定段階
  • Director 自動補完 — project.director 設定有 → 自動 participants 追加
  • Director 不在 enforce — 設定無 + mention 無 → HTTP 400 拒否 + 警告
  • PM 必須含む — QC = Director / Review = mention 主体 — どちらも PM 必須
  • task status → review patch — Calendar enum 'review' 自動 patch (殿御命 2026-06-08)
  • Push + SSE 配信 — 担当者 メンション気づき → msg 確認 → QC ビューアへ

🌸 使用例 — 架空 project「桜咲く街」

佐藤 User が桜咲く街 sq01 c05 v04 部内 Review OK 後、「🔍 QC」 選択 → 山田 Director 自動 mention → Director Dashboard 紫 card で気付き → qc_viewer で判定。

🛠 操作方法

  1. shot_detail で対象 asset card → 📤 既存 asset で QC 依頼 → modal で「🔍 QC」 選択
  2. (Director 設定有 project) mention 不要・直接送信
  3. (Director 設定無 project) ⚠️ 警告 → mention で代替担当 1 名以上選択
  4. 受信側: 通知 toast → SHOT thread → ▶ QC ビューア → Director 判定
